# Concept Method

## 1. Frame The Product Moment

Write a private sentence:

> When [user] opens this screen during [moment], they need to understand [truth] and confidently [action].

Convert it into a three-second contract containing no more than three answers.

## 2. Separate Product Structure From Visual Style

List:

- required information;
- required actions;
- existing navigation and platform conventions;
- emotional requirement;
- constraints from real data and implementation;
- current product strengths that should survive.

Do not begin with colors, glass, gradients, cards, or animation.

## 3. Build A Divergence Matrix

Generate candidates by crossing different choices from these axes:

| Axis | Examples |
|---|---|
| Information model | state, sequence, territory, relationship, transformation, comparison, conversation |
| Dominant artifact | instrument, material object, field, editorial composition, spatial environment, kinetic system |
| Spatial grammar | centered, edge-driven, layered depth, asymmetric split, continuous surface, progressive disclosure |
| Interaction | reveal, tune, assemble, focus, scrub, hold, draw, navigate, transform |
| Emotional posture | calm, decisive, protective, reflective, energetic, clinical, playful |
| Time behavior | static snapshot, accumulation, decay, phase shift, history trace, forecast |

Do not use a metaphor merely because it makes an attractive image. It must improve comprehension or action.

## 4. Require A Proprietary Functional Artifact

The selected concept should contain one memorable object or behavior that:

- encodes product data;
- supports the primary action;
- can change across real states;
- belongs specifically to this product;
- remains useful when decoration and animation are removed.

Examples of valid artifact classes include a domain-specific instrument, material system, adaptive spatial model, or branded transformation. Do not reuse these as concepts without grounding them in the product.

## 5. Score Candidates

Score each finalist from 1 to 10:

- immediate clarity;
- daily usefulness;
- originality;
- brand ownership;
- emotional fit;
- accessibility;
- responsive potential;
- technical feasibility.

Reject any candidate below 7 in clarity or usefulness, regardless of visual impact.

## 6. Make The Concept Decision-Complete

Specify:

- what appears first, second, and third;
- what the dominant artifact means;
- what happens on interaction;
- how empty and dense states behave;
- how reduced motion preserves meaning;
- how mobile changes the composition rather than shrinking it;
- what existing product behavior is retained, displaced, or removed.
